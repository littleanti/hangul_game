# PRD — 사자성어 음절 받아쓰기 (Idiom Syllable Typer)

> Product Requirements Document
> Last updated: 2026-06-10

## 1. 제품 개요

**제품명**: 사자성어 음절 받아쓰기 — 한자 어원을 보며 한 음절씩 받아쓰는 IME 비계 게임

**한 줄 요약**: S10(문해력 해독기)에서 의미를 익힌 사자성어 10개를 한자+한글독음 병기·TTS 제시로 다시 만나며, 4×1 슬롯에 한 음절씩 점진적 입력 방식으로 산출하는 IME 산출 비계 게임.

**위치**: 7단계 한글 학습 로드맵의 11번째 독립 브릿지 단계
(`10_literacy_decoder` → **`11_idiom_syllable_typer`** → `12_four-character_idiom_crossword`)

**배포 포트**: 4332

**배포 환경**: 단일 페이지 정적 웹앱 (모바일 / 태블릿 우선, PC 보조), PWA

## 2. 문제 정의 — 메워야 할 학습 갭

### 2.1 갭의 실체 (LEARNING_PROGRESSION_ANALYSIS.md §3.6·§5.2 인용)

S12(`12_four-character_idiom_crossword`)는 시리즈 전체에서 유일하게 **자유 한글 IME 타이핑**을 강제한다(`commitChar` U+AC00–D7A3, jamo 거부). 그런데 S1~S10은 한 번도 한글 음절을 *산출*하게 한 적이 없다 — 모든 입력이 탭/드래그로 주어진 타일 조립이었다. 분석 보고서는 이를 다음 두 가지 동시 벼랑으로 판정했다.

| 루브릭 차원 | 점수 | 도약 요지 |
|---|---|---|
| D3 입력 방식 | 5 | 자유 한글 IME 타이핑 강제 — 시리즈 원칙 '한글 IME 회피' 정면 위배 |
| D5 선행 기술 공백 | 5 | 한글 음절 IME 산출이 S1~S10 어디에서도 훈련된 적 없음 (무조건 벼랑) |

두 차원 모두 D=5(무조건 벼랑 기준)로 합산 27/평균 4.50을 기록, 시리즈 내 **가장 심각한 전이**로 판정됐다. 본 게임(`11_idiom_syllable_typer`)은 보고서 §5.2 처방을 따라 **D3(입력 패러다임)과 D5(IME 산출 선행기술)만** 격리하여 점진적으로 완충한다. 두 차원은 '한글 음절 조합'이라는 동일한 기저 기술에 뿌리를 두므로 단일 차원 완충 원칙(§1.4)을 위반하지 않는다.

### 2.2 S10이 제공하는 발판

S10(`문해력 해독기`)의 보스 스테이지는 아래 10개 사자성어를 형태소 조립(탭/드래그 타일)으로 노출했다. 학습자는 **의미와 한자 구조를 이미 안다** — 이것이 신규 어휘 부하를 0으로 묶는 발판이다.

| # | 사자성어 | 한자 | 의미 |
|---|---|---|---|
| 1 | 일석이조 | 一石二鳥 | 두 가지 이득 |
| 2 | 이심전심 | 以心傳心 | 마음이 통함 |
| 3 | 동문서답 | 東問西答 | 엉뚱한 대답 |
| 4 | 오리무중 | 五里霧中 | 갈피를 못 잡음 |
| 5 | 일취월장 | 日就月將 | 날로 발전함 |
| 6 | 청출어람 | 靑出於藍 | 제자가 스승보다 나음 |
| 7 | 천고마비 | 天高馬肥 | 가을 하늘과 살찐 말 |
| 8 | 화룡점정 | 畵龍點睛 | 마지막 마무리 |
| 9 | 대기만성 | 大器晚成 | 늦게 이루는 큰 인물 |
| 10 | 백발백중 | 百發百中 | 항상 맞힘 |

> **잔존 위험 명시**: S12의 어휘 풀은 ~100개 이상이다. 본 게임이 10개의 IME 산출을 숙달해도, S12의 나머지 ~90개 사자성어(의미·철자 미습득)는 별도 어휘 습득 처방이 필요하다. 이는 본 게임의 범위 밖이다(LEARNING_PROGRESSION_ANALYSIS.md §3.6·§4 참조).

## 3. 타겟 사용자

| 사용자 그룹 | 특징 | 주요 니즈 |
|---|---|---|
| 초6 학습자 (12세) | S10 보스를 탭/드래그로 통과한 직후 | 의미는 알지만 '직접 타이핑'이 낯섦 → 점진적 타이핑 비계 |
| 중1 학습자 (13세) | S12 크로스워드 진입 직전 | IME 산출 유창성 확보, 타이핑 불안 해소 |
| 학부모 / 교사 | 자녀의 한자어 표기 능력 진척도 확인 | 오답 어휘·반복 횟수 기록 |

### 3.1 인지 발달 정합

- **피아제 구체적 조작기 후기 ~ 형식적 조작기 진입(11~13세)**: 규칙 기반 추상 조작(한글 음절 조합 원리) 이해 가능. 단계별 페이딩이 비계로 적절히 작동한다.
- **Ehri 통합 알파벳 단계(consolidated alphabetic)**: 음절 청킹이 자동화된 상태이므로 IME로 전체 음절을 산출하는 과제가 ZPD 상단에 위치한다.
- **CLT(인지부하 이론)**: 3단 페이딩이 외재적 부하를 단계별로 낮추며, 신규 어휘 부하 0 정책으로 본유적 부하를 고정한다.

## 4. 학습 목표

### 4.1 획득 기술 (이 게임에서 배우는 것)

| # | 획득 기술 | 설명 |
|---|---|---|
| LO1 | 한글 음절 IME 산출 자동화 | 한글 자모 조합 원리를 익혀 음절 단위로 IME 입력 |
| LO2 | 인식 → 인출·표기 전이 | 의미·한자를 보고 떠올리는 수용에서 스스로 표기를 산출하는 능동적 인출로 전환 |
| LO3 | 3단 입력 페이딩 적응 | 음절블록 탭 → 초성힌트+자모 조립 → 자유 IME 순서로 비계가 줄어도 산출 유지 |
| LO4 | 사자성어 10개 표기 내재화 | S10에서 형태소 인식으로 습득한 10개 사자성어의 한글 음절 철자를 능동 인출 가능 |

### 4.2 비목표 — 인접 단계로 유보하는 것

다음 항목은 본 게임의 범위 밖이다. 아래 목록은 벼랑을 막기 위한 **의도적 제외**이다.

| 비목표 | 유보 단계 | 근거 |
|---|---|---|
| 크로스워드 공간 추론 (교차·Tab 순환 항법) | S12 | D2 인지 조작을 동시에 올리면 CLT 위반 |
| 사자성어 10개 이외 신규 어휘 습득 | S12 또는 별도 어휘 모듈 | 신규 어휘 부하 0 정책 (단일 차원 완충) |
| 메타포·비유적 의미 해석 심화 | S12 | D1/D6 건드림 → 단일차원 완충 원칙 위배 |
| 사자성어 작문 / 문장 만들기 | 별도 산출 단계 | 쓰기 산출 가닥 — 구조적 공백 §4 |
| 음성 인식 채점 (발음 평가) | 미래 로드맵 | Web Speech Recognition 정밀도 미확보 |
| 멀티플레이어 / 경쟁 모드 | 미래 로드맵 | 서버 의존성 없음 원칙 |

## 5. 핵심 게임 루프

```
[시작 화면]
    │
    ▼
[사자성어 제시]
  - 한자 4글자 (예: 一石二鳥)
  - 한글독음 힌트 병기 (예: 일석이조)
  - TTS 자동 발음 (ko-KR)
  - 의미 한 줄 설명 (예: "두 가지 이득")
    │
    ▼
[4×1 입력 슬롯]  ←────────────────────────────────┐
  ┌────┬────┬────┬────┐                             │
  │ 슬 │ 롯 │ 슬 │ 롯 │  ← 현재 활성 슬롯 하이라이트  │
  └────┴────┴────┴────┘                             │
    │                                               │
    ▼                                               │
[3단 페이딩 입력 방식 (슬롯별 독립 적용)]             │
  Lv.1 음절블록 탭                                  │
    : 하단 도크에 8개 후보 음절블록 제시              │
    : 정답 음절 탭 → 슬롯 채움                       │
  Lv.2 초성힌트+자모 조립                            │
    : 초성 힌트 표시 (예: 'ㅇ')                      │
    : 자모 키패드에서 조합 → 완성 음절 슬롯 채움       │
  Lv.3 자유 IME                                     │
    : 힌트 없음, 자유 타이핑                          │
    : S12 입력 방식과 동일                            │
    │                                               │
    ▼                                               │
[음절 채점]                                         │
  ✓ 정답 → 슬롯 녹색, 한자 어원 팝업 (해당 음절)     │
  ✗ 오답 → 슬롯 흔들림, 재시도 허용                  │
    │                                               │
    ▼                                               │
[4개 슬롯 모두 정답]                                │
  → 사자성어 전체 완성 애니메이션                    │
  → 다음 사자성어로 진행 ──────────────────────────┘
    │
    ▼
[10개 완료 → 완료 화면]
  - 맞힌 횟수 / 시도 횟수
  - 오답 목록 + TTS 복습
  - 다시 하기 / 리더보드 진입
```

### 5.1 3단 페이딩 세부 규칙

| 레벨 | 명칭 | 비계 | 진입 조건 |
|---|---|---|---|
| Lv.1 | 음절블록 탭 | 8개 후보 블록(정답 4개+방해 4개) 탭 선택 | 기본(첫 회차 또는 오답 후 재시도) |
| Lv.2 | 초성힌트+자모 조립 | 초성(첫 자모) 힌트 표시, 자모 키패드 조립 | 설정에서 선택 또는 Lv.1 전체 정답 후 해제 |
| Lv.3 | 자유 IME | 힌트 없음, 표준 IME 입력 | 설정에서 선택 또는 Lv.2 전체 정답 후 해제 |

> 페이딩 레벨은 사자성어 단위가 아닌 **슬롯(음절) 단위**로 적용된다. 학습자가 'ㄷ'을 반복 틀리면 해당 슬롯만 Lv.1로 재강화, 나머지 슬롯은 현 레벨 유지.

> 자동 페이딩 진급은 **기본 OFF**다. 설정에서 켠 경우에만 오답 없는 문항 완료 시 레벨이 자동 상승하며, OFF 상태에서는 홈에서 선택한 레벨로 고정 진행된다. 자동 진급 OFF가 곧 레벨 고정이므로 별도의 '레벨 고정' 옵션은 두지 않는다.

## 7. 콘텐츠 / 데이터 범위

### 7.1 핵심 어휘 데이터 (10개 — S10 보스 자산 재사용)

`10_literacy_decoder/src/data/idioms.js`의 `BOSS_IDIOMS` 배열을 직접 참조하거나 동일 구조로 복사하여 사용한다. 각 항목의 필드:

| 필드 | 설명 | 예시 |
|---|---|---|
| `word` | 한글 사자성어 | `"일석이조"` |
| `hanja` | 한자 배열 (4개) | `["一","石","二","鳥"]` |
| `meaning` | 한 줄 의미 | `"두 가지 이득"` |
| `hint` | 어원 힌트 문장 (Lv.1 도크 툴팁) | `"돌 하나로 새 두 마리를 잡아요"` |
| `contextStory` | 어원 팝업용 짧은 일화 | `"철수가 빨래를 …"` |

### 7.2 음절 분해 예시 (게임 문항 구체 예시)

| 사자성어 | 슬롯1 | 슬롯2 | 슬롯3 | 슬롯4 | Lv.1 방해 음절 예시 |
|---|---|---|---|---|---|
| 일석이조 | 일 | 석 | 이 | 조 | 월, 삭, 의, 도 |
| 동문서답 | 동 | 문 | 서 | 답 | 통, 분, 셔, 탑 |
| 화룡점정 | 화 | 룡 | 점 | 정 | 하, 용, 점, 장 |
| 청출어람 | 청 | 출 | 어 | 람 | 정, 촐, 으, 남 |
| 천고마비 | 천 | 고 | 마 | 비 | 전, 교, 나, 배 |

> 방해 음절은 정답 음절과 초성 또는 모음 1자만 다른 유사 음절로 구성해 변별력을 훈련한다.

### 7.3 TTS 발화 시점

| 시점 | 발화 내용 |
|---|---|
| 문항 첫 제시 | 사자성어 전체 (`ko-KR` Web Speech API) |
| TTS 버튼 탭 | 사자성어 전체 재발화 |
| 정답 슬롯 채움 | 해당 음절 1개 발화 |
| 어원 팝업 표시 | 한자 음·뜻 발화 (선택) |
| 완료 화면 오답 복습 | 오답 사자성어 발화 |

## 11. 시리즈 연속성 요구

### 11.1 이전 단계 (S10 — 문해력 해독기) 와의 연결

| 연결 항목 | 구체 내용 |
|---|---|
| 어휘 자산 재사용 | `10_literacy_decoder/src/data/idioms.js`의 `BOSS_IDIOMS` 10개 전량 재사용 (신규 어휘 0) |
| 한자 메타 재사용 | `hanja` 배열, `contextStory`, `meaning`, `hint` 필드 그대로 활용 |
| 의미 선행 가정 | 학습자가 S10 보스 스테이지 통과 → 10개 사자성어 의미·한자 구조 기지식으로 전제 |
| 난이도 연속성 | S10 보스는 탭/드래그 형태소 조립(수용) → S11은 음절 IME 산출(능동). 인지 조작 1칸 전진 |

### 11.2 다음 단계 (S12 — 사자성어 크로스워드) 와의 연결

| 연결 항목 | 구체 내용 |
|---|---|
| IME 입력 선행 | S11 Lv.3 자유 IME가 S12 입력 방식(`commitChar` U+AC00–D7A3)과 동일 패러다임 — 직접 진입로 |
| 어휘 발판 | S11에서 IME 표기를 내재화한 10개가 S12의 초기 어휘 풀 역할 |
| 비유보 항목 | 크로스워드 공간 추론(교차 슬롯·Tab 순환 항법)은 본 게임에서 도입하지 않음 — S12에 유보 |
| 데이터 스키마 호환 | localStorage 키는 `s11_` 접두사 사용. S12가 읽을 수 있도록 완료한 사자성어 목록을 공유 스키마(`learner.completedIdioms`)에 기록 권장 |

### 11.3 잔존 갭 (S11이 메우지 못하는 것 — 명시적 공백)

- **S12 신규 어휘(~90개+)**: S11이 다루는 10개 외의 사자성어는 S12 내에서 별도 어휘 습득 경로가 필요하다.
- **크로스워드 공간 항법**: 교차 셀 탐색, Tab 순환, 단어 방향 인식은 S12의 독립 학습 목표다.
- **고전 전고 비유 해석**: 와신상담·각주구검 등 표면형 추론 불가 사자성어의 비유적 의미는 S12가 담당한다.

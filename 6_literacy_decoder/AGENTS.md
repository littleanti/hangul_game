# 6_literacy_decoder — 문해력 해독기 게임

## Purpose
단어 단위 조립을 넘어 **문장과 문단** 속에서 미지의 한자어를 형태소 지식과 문맥 단서로 추론하는 최종 브릿지 게임. 본 단계를 완료한 학습자는 사자성어 크로스워드(`../7_four-character_idiom_crossword/`)의 비유적·은유적 4글자 압축 의미 해석에 진입할 준비를 갖춘다.

## Target & Cognitive Goal

| 항목 | 내용 |
|------|------|
| 대상 연령 | 초등 5 ~ 6학년 (확장: 중학 1학년) |
| 발달 단계 | 문맥 추론 + 형태 의미론적 통합 |
| 핵심 인지 목표 | 미지 어휘의 뜻을 형태소 + 문맥으로 유추, 직독직해 자동화 |
| 선행 게임 | `../5_vocabulary_tree/` — 형태소 파생 어휘망 보유 |
| 후행 게임 | `../7_four-character_idiom_crossword/` — 비유적 4자성어 진입 |

## Game Mechanics

### 핵심 루프 (해독기 모드)
1. 학년 수준에 맞는 흥미로운 지문 제시 (전래동화, 과학 단편, 역사 일화)
2. 핵심 한자어가 **빈칸** 또는 **암호화 기호**로 표시 (예: `옛날 옛적, 한 [□□]가 살았는데...`)
3. 화면 측면에 후보 한자 형태소들이 흩어져 표시 (5단계까지 학습한 한자 풀)
4. 학습자는 문맥을 읽고 알맞은 한자 조합 선택 → 단어 완성
5. 완성된 단어가 지문에 채워지며 의미 통합 확인
6. 정답 시 어원 풀이 팝업: "`農夫` = 농(農: 농사) + 부(夫: 사내) → 농사 짓는 사람"

### 사자성어 진입 훈련 (보스 스테이지)
지문이 특정 상황을 묘사할 때, 4개의 한자 형태소를 힌트로 던지고 사자성어를 추론하게 함:

> 짧은 이야기: "철수가 영희에게 '오늘 점심 뭐 먹을래?'라고 물었더니, 영희는 '내일 비가 올 것 같아'라고 답했다."
> 힌트 형태소: 東(동/동쪽) · 西(서/서쪽) · 問(문/물을) · 答(답/답할)
> 추론 정답: **동문서답(東問西答)** — "동쪽을 물으니 서쪽을 답한다"

이 보스 스테이지가 곧 `../7_four-character_idiom_crossword/`로 이어지는 게이트웨이.

### 응용 작문 미션 (선택)
- 지문에서 만난 한자어를 사용해 짧은 문장 짓기
- AI 또는 키워드 매칭으로 사용 적합성 평가 (느슨한 평가 — 시도 자체를 보상)

### 난이도 곡선
| 단계 | 지문 길이 | 빈칸 수 | 형태소 힌트 |
|------|----------|---------|-----------|
| 입문 | 100자 내외 | 1 | 음·뜻 모두 표시 |
| 중급 | 300 ~ 500자 | 2 ~ 3 | 뜻만 표시 |
| 심화 | 800자 이상 | 4 ~ 5, 사자성어 포함 | 형태소만 표시 |

## Mobile-First Considerations

본 게임은 **긴 지문 + 빈칸 + 형태소 힌트**의 3요소를 동시에 다루므로 모바일 화면에서 가장 까다롭다.

| 항목 | 권장 사양 |
|------|----------|
| 화면 모드 | **세로(Portrait) 권장** — 지문은 수직 가독성 우선 |
| 폰트 크기 | 본문 **18px 이상**, 빈칸 강조 색상 |
| 형태소 힌트 위치 | 상단 사이드바가 아닌 **하단 sticky 도크** |

### 폰 vs 태블릿 레이아웃 분기

#### 폰 (≤ 600px width)
- 지문은 **단락별 페이지네이션** — 800자 지문은 4 ~ 5 페이지로 분할
- 현재 단락 + 빈칸이 항상 화면 상단 절반에 위치
- 형태소 후보는 하단 도크에서 가로 스크롤
- 빈칸 탭 시 도크가 확장되어 후보 선택 → 선택 후 자동 닫힘

#### 태블릿 (≥ 768px width)
- 좌측 65%: 지문 (전체 표시 가능)
- 우측 35%: 형태소 힌트 풀 + 어원 풀이 패널
- 빈칸 ↔ 형태소 카드 드래그 매칭

### 가독성 / 접근성
- **폰트 크기 조절** 설정 (16 / 18 / 22px) — 학년·시력 차이 대응
- 한자에는 항상 **루비 텍스트** 또는 클릭 시 음·뜻 툴팁 (한자 시각만 노출 금지)
- **다크 모드** 지원 — 야간 학습 부담 감소
- TTS로 지문 전체 읽어주기 (자동 재생 X, 사용자가 명시적으로 시작)

### 성능
- 지문 코퍼스는 **JSON 분할 로드** (학년군별 chunked)
- 어원 풀이 팝업은 lazy 로드 — 정답 시점에서만 fetch
- 형태소 힌트 풀이 많아질수록 가상 스크롤 적용

## For AI Agents

### Working In This Directory
- 미구현 설계 단계
- 권장 스택: Vanilla JS + CSS, 포트 **4326**
- 5단계의 한자 DB와 학습 진척도(IndexedDB) 공유 또는 호환 형식 사용

### Implementation Priorities
1. **지문 코퍼스** — 학년군별 지문 100 ~ 200편 (저작권 free 또는 자체 작성)
2. **빈칸 자동 추출** — 지문에서 학습 대상 한자어를 마킹하는 메타데이터 시스템
3. **형태소 힌트 풀 생성** — 정답 + 오답 형태소를 적절히 섞는 디스트랙터 알고리즘
4. **사자성어 보스 스테이지** — `7_four-character_idiom_crossword`의 데이터(`data.js`)와 연동 가능한 형식 설계
5. **읽기 속도/정답률 측정** — 직독직해 자동화 진척도 시각화
6. **반응형 레이아웃** — 폰 페이지네이션 / 태블릿 분할 화면 자동 분기

### Key Behaviors to Preserve
- 지문은 **흥미 우선** — 아무리 교육적이어도 지루하면 학습 효과 0
- 빈칸은 학습자가 **추론할 수 있는 단서**가 지문에 반드시 존재해야 함 (불가능한 추론 금지)
- 오답에 대한 피드백은 **형태소 단위 분해 설명** ("이 한자는 '물 수'인데, 문맥은 '불'에 관한 이야기네요")

### Testing Requirements
- 추론 가능성 검증 — 지문만 읽고 빈칸이 풀리는지 사용자 테스트
- 사자성어 보스 스테이지를 통과한 학습자가 `7_four-character_idiom_crossword`에서 평균 정답률 향상 검증
- 폰 / 태블릿 레이아웃 분기 검증

# 📋 PRD — 문해력 해독기 (Literacy Decoder)

> Product Requirements Document
> Last updated: 2026-04-25

## 1. 제품 개요

**제품명**: 문해력 해독기 — 지문 속 한자어를 형태소와 문맥으로 추론하는 게임

**한 줄 요약**: 초등 고학년 학습자가 흥미로운 지문 속 빈칸(미지 한자어)을 **문맥 단서 + 형태소 지식**으로 해독하며 직독직해를 자동화하는 웹 기반 학습 게임.

**위치**: 7단계 한글 학습 로드맵의 6단계 (`5_vocabulary_tree` → `6_literacy_decoder` → `7_four-character_idiom_crossword`)

**배포 환경**: 단일 페이지 정적 웹앱 (모바일 / 태블릿 우선, PC 보조)

## 2. 타겟 사용자

| 사용자 그룹 | 특징 | 주요 니즈 |
|---|---|---|
| 초5 ~ 초6 학습자 | 1자→N어휘 파생 경험 보유 (5단계 졸업) | 문맥 속 미지 어휘 추론, 직독직해 자동화 |
| 중1 (확장) | 문해력 격차로 교과 따라가기 어려움 | 한자어 분해 능력으로 교과 어휘 학술화 |
| 학부모 / 교사 | 자녀의 독해력 진척도 확인 필요 | 정답률 / 읽기 속도 시각화 |
| 한국어 학습자 (외국인 중급+) | 한자어 비중이 높은 한국어 텍스트 어려움 | 형태소 분해로 어휘 폭발 진입 |

## 3. 사용자 시나리오

### S1. 학습자가 혼자 한 단원 진행
- 학교에서 돌아와 태블릿 세로 모드로 앱 실행
- "오늘의 지문" — 300자 분량 전래동화 일화 제시
- 빈칸 `[□□]` 두 곳 (예: `옛날, 한 [□□]가 살았는데...`)
- 하단 도크에서 형태소 후보 6개 중 정답 조합 (`農 + 夫`)을 빈칸으로 끌어옴
- 정답 시 어원 풀이 팝업: "농(農: 농사) + 부(夫: 사내) → 농사 짓는 사람"
- 지문이 완성되며 다음 페이지 이동

### S2. 사자성어 보스 스테이지
- 짧은 일화 지문이 끝나면 4개 형태소 힌트 등장 (`東·西·問·答`)
- "이 상황을 사자성어로 표현하면?" — 학습자가 형태소를 4×1 격자에 배치
- 정답 `동문서답(東問西答)` 도달 시 7단계 게임 진입 권한 부여
- 이 보스 통과는 7단계 (`four-character_idiom_crossword`)의 초기 어휘 풀로 자동 등록

### S3. 학부모가 진척도 확인
- 메뉴 → "학습 기록" 진입
- 주간 읽기 속도 / 정답률 그래프 확인
- 자주 틀린 한자어 리스트 + 어원 복습 카드 제공

### S4. 응용 작문 미션 (선택)
- 지문에서 만난 한자어 (`農夫`)로 짧은 문장 짓기
- 키워드 매칭으로 사용 적합성 평가 (느슨한 평가, 시도 자체를 보상)

## 4. 기능 요구사항

### P0 (필수)
| ID | 기능 | 설명 |
|---|---|---|
| F1 | 시작 화면 | 학년 / 난이도 선택, 이어하기, 학습 기록 진입 |
| F2 | 지문 렌더 | 지문 텍스트 + 빈칸 마커 표시, 페이지네이션 (모바일) |
| F3 | 빈칸 인터랙션 | 빈칸 탭 → 활성화, 형태소 후보 도크 강조 |
| F4 | 형태소 도크 | 정답 + 디스트랙터 형태소 카드, 음·뜻 표시 |
| F5 | 단어 조립 | 형태소 카드 드래그/탭 매칭 → 빈칸에 단어 완성 |
| F6 | 채점 & 어원 풀이 | 정답 시 어원 분해 팝업, 오답 시 형태소 단위 피드백 |
| F7 | 진척도 / 다음 지문 | 진행바, 다음 지문 자동 진입, 세션 종료 화면 |

### P1 (확장)
| ID | 기능 | 설명 |
|---|---|---|
| F8 | 사자성어 보스 스테이지 | 4개 형태소 힌트 → 사자성어 추론, 7단계 게이트웨이 |

| F14 | 다크 모드 | 야간 학습 부담 감소 |
| F15 | localStorage / IndexedDB 영속화 | 누적 학습 진척도, 5단계 데이터 호환 스키마 |

### P2 (향후)
- **응용 작문 미션** — 학습한 한자어로 문장 짓기, 키워드 매칭 평가
- **PWA 변환** — 홈 화면 설치, 오프라인 캐시
- **음성 답안** — 빈칸 음성으로 채우기 (Web Speech Recognition)
- **친구와 경쟁** — 동일 지문 정답 시간 랭킹 (선택, 비식별)
- **학부모 / 교사 알림** — 주간 리포트 푸시 (PWA 알림)

## 5. 비기능 요구사항

| 항목 | 요구사항 |
|---|---|
| 성능 | 지문 페이지 전환 < 400ms, 빈칸 인터랙션 60fps |
| 호환성 | Chrome / Safari / Edge 최신, iOS Safari 15+, Android Chrome 100+ |
| 반응형 | 폰 (≤ 600px) / 태블릿 (≥ 768px) 자동 분기, 320px 최소 폭 지원 |
| 접근성 | 터치 타겟 최소 44dp, 본문 18px 이상, WCAG AA 색 대비 |
| 가독성 | 한자에는 항상 음·뜻 접근 수단 제공 (루비 또는 툴팁) |
| 오프라인 | 초기 로드 후 지문 코퍼스 + 한자 DB 캐시 사용 가능 |
| 데이터 | 누적 학습 데이터는 IndexedDB (localStorage 5MB 한도 회피) |

## 6. 성공 지표

- **추론 가능성**: 학습자가 지문만 읽고 빈칸을 풀 수 있는 비율 60% 이상 (사용자 테스트)
- **세션 지속**: 평균 세션 시간 10 ~ 15분, 1세션 당 최소 1지문 완주율 70%
- **사자성어 게이트웨이 효과**: 보스 스테이지 통과 학습자가 7단계 진입 시 평균 정답률 향상 (대조군 대비 +15%)
- **재방문**: 7일 누적 사용 사용자 30% 이상
- **읽기 속도 향상**: 동일 학습자의 4주 후 읽기 속도(자/분) 20% 향상

## 7. 범위 제한 (Out of Scope)

- 회원가입 / 로그인 — 모든 학습 데이터는 로컬 (IndexedDB)
- 외부 서버 데이터 전송 — 텔레메트리 옵트인 외 송신 없음
- 멀티플레이어 실시간 대전
- 백엔드 / DB
- 한자 쓰기 입력 (4단계 원칙 계승: 한자 자체 암기 강요 금지)
- AI 평가 (응용 작문은 P2이며 키워드 매칭 수준)

## 8. 의존성 / 통합 지점

| 구분 | 대상 | 호환 형식 |
|---|---|---|
| 선행 | `5_vocabulary_tree` | 한자 마스터 DB, 학습 진척도 IndexedDB 스키마 |
| 후행 | `7_four-character_idiom_crossword` | `data.js` `SAJASUNGO_DATA` 배열 (`{ word, meaning, hint }`) |
| 공통 | 학부모 대시보드 (Future) | 학습자 ID + 단계별 진척도 통합 |

## 9. 오픈 이슈

- [ ] 지문 코퍼스의 저작권 처리 — 자체 작성 vs 저작권 free 전래동화 발췌 비율
- [ ] 4단계 / 5단계의 구체적 데이터 포맷 — 두 게임이 미구현 단계이므로 본 게임이 **출발점 스키마**를 제안하고 후행 호환 보장
- [ ] 사자성어 보스 스테이지 정답 인정 범위 — 4글자 순서 고정 vs 형태소 집합만 일치
- [ ] 학습 기록 시각화 라이브러리 선택 (Vanilla SVG vs Chart.js)
- [ ] PWA Service Worker 도입 시점 (MVP 후 vs MVP 동시)
- [ ] 모바일 지문 페이지네이션 알고리즘: 글자 수 기준 vs 단락 기준 vs 빈칸 단위 기준

## 10. 홈·설정·완료 화면 디자인 공통 규격

시리즈 전체의 통일감을 위해 **홈 화면(시작 화면)**, **설정 화면**, **게임 완료 화면(end-screen)** 은 `1_chosung_quiz` 의 디자인 시스템을 기준으로 제작한다.

| 디자인 요소 | 규격 |
|---|---|
| 제목 폰트 | **Jua** (Google Fonts) |
| 설명·본문 폰트 | **Gowun Dodum** (Google Fonts) |
| 버튼 디자인 | 1단계 `components.css` 버튼 스타일 — 큰 라운드 버튼, CSS 변수 색상 |
| 색감 | `1_chosung_quiz/src/css/tokens.css` CSS 변수와 동일한 팔레트 사용 |
| 배경 | `--color-bg` 등 공통 토큰과 동일한 배경 색상 |

> 지문 읽기 화면·대시보드 등 게임 고유 콘텐츠 화면은 각 게임 특성에 맞게 확장 가능하다.  
> 단, 시작·설정·완료 화면은 공통 디자인 언어를 유지한다.

## 2. 문제 정의 — 메워야 할 학습 갭

### 2.1 한글 해득 위계의 '문장' 공백

한글 해득의 표준 발달 위계는 **자모 → 음절 → 낱말 → 문장 → 글**이다. 8단계(어휘력 세계수)를 마친 학습자는 파생어·한자어 ~800어휘를 분해·유추할 수 있지만, 낱말을 문장 맥락 안에서 처리하는 훈련은 아직 받은 적이 없다. 10단계(문해력 해독기)가 요구하는 "100~300자 지문 유창 독해 + 문맥단서 식별"은 이 경험 없이는 도달할 수 없는 벼랑이다.

`LEARNING_PROGRESSION_ANALYSIS.md §5.1`은 이 구간의 D5(선행 기술 공백) 점수를 5(무조건 벼랑)로 판정하고, 단문 클로즈 워밍업 `5.5_sentence_clue_garden`(현재 폴더 `9_sentence_clue_garden`으로 구현)을 **필수(必) 독립 단계**로 권고했다. 본 게임은 그 처방을 구현한다.

> "S5에서 습득한 한자어를 1~2문장(20~60자) 안에 놓고 주변 문맥 단서로 빈 어휘 1개를 채우는 단문 클로즈 워밍업. S5의 child-friendly 예문 자산을 학습자 수행형 문맥추론으로 전환."
> — LEARNING_PROGRESSION_ANALYSIS.md §5.1

### 2.2 단일 차원 완충 원칙

`LEARNING_PROGRESSION_ANALYSIS.md §1.4`의 가교 단계 설계 원칙에 따라 본 게임은 **D5(문맥단서 식별·단문 독해)와 D6(낱말→문장 인접 한 칸 이동)만** 전진시키고, 나머지 차원은 8단계 수준으로 동결한다.

- D1(어휘 난이도): 8단계 한자어 수준 동결
- D2(인지 조작): 인출·선택 수준 유지 (분석/산출로 상승 없음)
- D3(입력 방식): 8단계와 동일한 탭/드래그 도크 조립 불변
- D4(대상 연령): 초4~5(만 9~11세), 8단계 상단과 연속

## 3. 대상 사용자

| 사용자 그룹 | 특징 | 주요 니즈 |
|---|---|---|
| 초등 4~5학년 (만 9~11세) | 8단계 어휘 세계수 수료, 한자어 기초 어휘 ~800개 보유 | 낱말 지식을 문장 읽기로 연결하는 첫 경험 |
| 초등 고학년 진입 직전 학습자 | 단어 분해 능숙, 글 독해 경험 부족 | 쉬운 단문에서 문맥 단서 사용법 습득 |
| 부모·교사 | 아이의 어휘→독해 전환 단계 지원 | 난이도·힌트 조절, 학습 기록 확인 |

### 3.1 인지 발달 정합

초4~5(만 9~11세)는 Piaget의 구체적 조작기 후반에서 형식적 조작기 초입으로, 맥락적 추론(문맥 읽기)이 발달하는 시기다. 낱말 단위 처리에서 문장 단위 처리로 확장되는 Chall 읽기 발달 Stage 2 후반에 해당하며, 이 연령대는 비계(힌트)를 제공하면 문맥 추론을 시도하고 적차 내면화할 수 있다.

8단계의 대상 연령(초3~4, 만 8~10세)과 자연스럽게 이어지며 연령 단조성(monotonicity)을 유지한다.

## 5. 핵심 게임 루프

```
[홈 화면]
   ↓ 시작 / 레벨 탭
[설정 화면] ──→ 난이도·문제수·힌트·TTS·효과음·이름
   ↓
[플레이 화면]
   문장 카드 — 1~2문장, 빈칸 [ ] 1개 표시
   ↓
   하단 도크 — 정답 1개 + distractor 2~3개 (칩 형태)
   ↓
   탭 또는 드래그+스냅 → 빈칸 채우기
   ↓
   정답 판정 (500ms 내 자동) → mint/red 피드백 + TTS + 효과음
   ↓
   1.5초 후 다음 문제 ──→ (마지막 문제 후)
   ↓
[완료 화면]
   점수·정답률·오답 목록·다시하기·리더보드
   ↓
[리더보드 화면] (선택)
```

### 5.1 3단 단서 페이딩

힌트 버튼(설정 ON 시 표시)을 탭할 때마다 단계 상승:

| 단계 | 표시 내용 | 설명 |
|---|---|---|
| 0 (초기) | 단서 없음 | 문장만 표시 |
| 1 | 음뜻 라벨 + 노란 하이라이트 | 해당 구절 위에 "기쁠 희(喜) — 즐거운 느낌" 형식의 라벨 + 노란 배경 |
| 2 | 하이라이트만 | 라벨 숨김, 노란 배경 유지 |
| 3 | 단서 완전 제거 | 하이라이트 제거, 힌트 버튼 비활성화 |

힌트 사용 단계(`hintLevelUsed`)는 세션 기록에 저장된다.

### 5.2 선택 취소

빈칸에 배치한 칩을 다시 탭하면 도크로 반환된다. 정답 판정이 시작된 후에는 취소 불가.

## 7. 콘텐츠 / 데이터 범위

### 7.1 문제 구성 원칙

- **문장 길이**: 1~2문장, 20~60자
- **빈칸**: 항상 1개 (`[ ]` 마커, 다중 빈칸 항목은 유효성 검사에서 거부)
- **어휘 출처**: 8단계(8_vocabulary_tree) 한자어 및 파생어 (신규 어휘 없음)
- **선택지**: 정답 1개 + 근접 의미 distractor 2~3개 (총 3~4개)
- **난이도 분포**: easy 최소 15개 / medium 최소 15개 / hard 최소 10개

### 7.2 문제 예시

| 난이도 | 문장 | 빈칸(정답) | 선택지 (distractor) | 단서 구간 |
|---|---|---|---|---|
| easy | 강아지가 [ ] 꼬리를 흔들며 반겼다. | 기쁘게 | 슬프게, 무섭게, 지루하게 | "꼬리를 흔들며 반겼다" |
| easy | 비가 오는 날 할머니는 [ ] 목소리로 이야기를 해주셨다. | 따뜻한 | 차가운, 날카로운, 조용한 | "이야기를 해주셨다" |
| medium | 등산을 마친 뒤 마신 물은 [ ]하기 어려울 만큼 시원했다. | 표현 | 상상, 설명, 이해 | "마신 물은", "시원했다" |
| medium | 그 선수는 오랜 [ ] 끝에 마침내 금메달을 목에 걸었다. | 노력 | 휴식, 여행, 방해 | "마침내 금메달을" |
| hard | 친구의 [ ] 있는 조언 덕분에 나는 중요한 결정을 바꿀 수 있었다. | 가치 | 문제, 재미, 불편 | "중요한 결정을 바꿀 수 있었다" |

### 7.3 데이터 스키마 (핵심 필드)

```js
{
  id: 'scg_001',          // 접두사 'scg_'
  sentence: '강아지가 [ ] 꼬리를 흔들며 반겼다.',
  answer: '기쁘게',
  choices: ['기쁘게', '슬프게', '무섭게', '지루하게'],
  hint: {
    level1: { label: '기쁠 희(喜) — 즐거운 느낌', highlight: [5, 18] },
    level2: { highlight: [5, 18] }
    // level3: 없음 (단서 완전 제거)
  },
  source: 'S8',           // 8단계 어휘 연계
  difficulty: 'easy'
}
```

## 9. 시리즈 연속성

### 9.1 8단계 (8_vocabulary_tree) ← 이전

| 연결 항목 | 내용 |
|---|---|
| **어휘 계승** | 8단계 한자어·파생어를 문장 안에 배치. `answer`·`choices` 어휘는 S8 어휘 DB와 동일 ID 체계 권장 |
| **입력 방식 계승** | 8단계의 탭/드래그 도크 조립 방식을 그대로 유지 (학습 부하 0) |
| **난이도 연계** | S8 easy 어휘 → S9 easy 문제 출제. S8 medium → S9 medium 순으로 난이도 매핑 |
| **디자인 계승** | 동일 디자인 시스템(tokens.css, components.css), 동일 폰트·색상 토큰 |

### 9.2 10단계 (10_literacy_decoder) → 다음

| 연결 항목 | 내용 |
|---|---|
| **독해력 발판** | S9 hard 정답률 기준점 도달 시 S10 시작 난이도와 연결 고려 |
| **스토리지 핸드오프** | IndexedDB `sessions` 데이터(`9scg_db`)가 S10에서 참조 가능한 포맷으로 설계 (`difficulty`, `accuracy` 필드 포함) |
| **스코프 차이** | S9는 단문(1~2문장, 빈칸 1개, 통사 동결) / S10은 100~300자 지문, 다문장, 문맥단서 통합. S9는 S10을 위한 단계적 준비 |

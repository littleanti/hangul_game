## M3 — 페이딩·난이도 시스템

### 1. 가로(landscape) 모드에서 도크가 화면 밖으로 밀려 스크롤 없이는 선택지가 보이지 않음 — ✅ 해결

- **심각도**: major
- **증상**: 뷰포트 800×420(가로)에서 `#play-screen.active`에 2-column 그리드는 적용되지만,
  `.play-controls`는 카드 옆(y=139)에 놓이는 반면 `#dock`은 카드 행 아래의 새 행(y=438)에
  배치되어 뷰포트 높이(420px)를 초과 — 선택지 칩이 전부 폴드 아래로 밀려 window 스크롤
  없이는 접근 불가. 우측 컬럼은 힌트 버튼 44px 외 대부분 빈 공간.
- **원인**: `game.css` 가로 모드 미디어쿼리에서 `.sentence-card`에 `grid-column: 1`만 지정하고
  **행 스팬이 없었음**. 그리드 자동 배치(row flow) 규칙상 `.sentence-card`(col1)와
  `.play-controls`(col2)가 같은 행에 놓인 뒤, `#dock`(col2)은 해당 행의 col2가 이미 차 있어
  '다음 행'으로 밀린다. 그런데 그 행의 높이는 카드 높이(285px)로 결정되므로 도크가
  카드 전체 높이만큼 아래(y=438)에서 시작하게 됨.
- **해결**: `.sentence-card`에 `grid-row: span 2`를 부여해 카드가 우측 컬럼의 두 행
  (`.play-controls` 행 + `#dock` 행)을 세로로 스팬하도록 수정. 이로써 도크가 힌트 버튼
  바로 아래, 카드 옆 우측 컬럼에 나란히 배치된다.
  - 수정 파일: `src/css/game.css` — 가로 모드 미디어쿼리 내 `.sentence-card` 규칙
  - 검증(Playwright, 뷰포트 800×420, 실제 플레이 화면): `#dock` y=232(카드 y=139 옆 우측
    컬럼, 수정 전 y=438), 도크 칩 4개 모두 bottom ≤ 372 < 420 — 스크롤 없이 전부 표시.
    세로(portrait) 모드는 미디어쿼리 외부라 영향 없음.

# PRD — 합성어 쪼개기 (5_compound_split)

> Product Requirements Document
> Last updated: 2026-06-11

---

## 1. 제품 개요

**제품명**: 합성어 쪼개기 — 낱말을 조각으로 나눠요

**한 줄 요약**: 만 6~7세 아동이 이미 알고 있는 순우리말 합성어를 탭 한 번으로 두 조각으로 쪼개며 "낱말은 뜻 있는 더 작은 부품으로 이루어진다"는 형태소성 통찰을 친숙한 재료로 선점화하는 웹 기반 학습 게임.

**배포 환경**: 단일 페이지 정적 웹앱 (모바일·태블릿 우선), 개발 포트 **4329**

## 2. 문제 정의 및 학습 갭

### 2.1 메우는 갭 (LEARNING_PROGRESSION_ANALYSIS.md §5.5·§3.3 인용)

3→4 전이(순우리말 어휘망 → 형태소 탐정)는 CLT 동시 상승 판정을 받은 **갭큼** 전이다(합산 20, D3=4·D6=4 동시 ≥4, CLT 트리거). 이 두 도약 중 본 게임이 담당하는 것은 **D6 단 하나**다.

| 도약 차원 | 수치 | 담당 |
|---|---|---|
| D3 입력 방식 | 4 — 돋보기·핀치/팬·가로 전환 | Stage 4 Level 0 In-stage 온보딩(별도 처방) |
| D6 콘텐츠 도메인 | 4 — 통째 읽기→형태소 인식 가닥 전환 개시 | **본 게임(5_compound_split)** |

핵심 통찰: "낱말은 뜻을 지닌 더 작은 조각(형태소)으로 이루어진다"(D6)라는 재범주화를 친숙한 순우리말 합성어로 미리 점화하면, Stage 4(형태소 탐정)에서 낯선 한자 기호·돋보기 UI와 형태소 개념을 동시에 습득해야 하는 이중 부하를 해소할 수 있다.

발달 근거: 형태론적 인식은 모국어 **합성어**에서 가장 먼저 발달한다(Carlisle; Kuo & Anderson — 합성어→굴절→파생 순서). 만 6~7세는 합성어 분해 인식의 임계 발달 창이다.

### 2.2 단일 차원 완충 원칙 준수 (§1.4)

본 게임은 D6만 완충한다. D1(어휘)·D2(인지 조작)·D3(입력)·D4(연령)는 Stage 3 수준에서 **동결**한다. D3의 돋보기·가로 UI는 Stage 4 첫 장면 온보딩이 담당하며, 두 처방이 합산되어야 3→4 전이가 완성된다.

## 3. 대상 사용자 및 연령·인지 발달 정합

| 사용자 그룹 | 연령 | 발달 특성 |
|---|---|---|
| 주 대상 — Stage 3 졸업 아동 | 만 6~7세 | Piaget 전조작기 후기~구체적 조작기 초기; 합성어 형태소 인식 발달 창(Carlisle); 순우리말 합성어 수용 어휘 보유 |
| 부모·보호자 | 성인 | 세팅 도우미 역할; 별도 교육적 개입 불필요한 수준의 단순 조작 |

**연령 정합 근거**: 만 6~7세는 Stage 3 상단(7세)과 Stage 4 하단(7세)의 중첩점이다(§5.5). 연령 단조성 위반 없음. Piaget 전조작↔구체적 조작 경계를 무리하게 횡단하지 않는다(D4 동결).

**선행 기술 전제**: Stage 3(순우리말 어휘망) 졸업 — 도입 세트 6개 합성어를 통째 단어로 이미 자동 해독 가능. 드래그/탭 협응(56dp) 보유.

## 4. 학습 목표 (획득 기술)

### 4.1 목표

| # | 획득 기술 | 설명 |
|---|---|---|
| G1 | **형태소성 통찰 점화** | 친숙한 순우리말 합성어가 뜻 있는 두 조각으로 나뉜다는 사실을 경험적으로 인식한다 |
| G2 | **공유 형태소 재인** | 동일 조각(방울·빛·송이)이 서로 다른 합성어에 반복 등장함을 알아챈다(단, 명시적 일반화는 Stage 4로 유보) |
| G3 | **"뜻 있는 최소 단위" 직관** | 잘못 자른 조각에는 그림이 뜨지 않는다는 피드백으로 형태소 정의(뜻 있는 최소 단위)를 암묵 습득한다 |

| Sino-Korean 파생 형태소·한자어 접사 | Stage 4·5 | 본 게임은 자립·투명 합성어만 사용 |

## 5. 핵심 게임 루프

```
[시작 화면]
     │ 시작 버튼
     ▼
[게임 플레이 화면] ─────────────────────────────────────────────┐
     │                                                          │
     │  1. 합성어 카드 1장 표시 (TTS 자동 재생)                    │
     │     └ 2모드(연습하기·도전하기)에 따른 경계 가시성·뜻 단서량 설정 │
     │                                                          │
     │  2. 아동이 카드를 탭(또는 분절 경계 영역 드래그)             │
     │     ├ 올바른 위치 탭 → 두 조각으로 분리 + 각 조각 그림·뜻 팝업 + 효과음·TTS
     │     └ 잘못된 위치 탭 → 그림 미출현 + "그 조각은 뜻이 없네" 메시지 + 부드러운 복귀
     │                                                          │
     │  3. 정답 피드백 확인 (조각 그림 2장 + 뜻 라벨) → 다음 카드  │
     │                                                          │
     │  6라운드 완료(도입 세트 전체) 또는 설정 문항 수 도달         │
     └──────────────────────────────────────────────────────────┘
     │
     ▼
[완료 화면]
     │ 다시 하기 / 홈으로
     ▼
[홈 화면]
```

**2모드 페이딩 (경계 가시성·뜻 단서량만 조절 — 입력·화면·어휘 불변)**

| 모드 | 경계 표시 | 뜻 단서 |
|---|---|---|
| 연습하기 | 점선 경계 | 한쪽(첫 번째 조각) 그림만 표시 |
| 도전하기 | 경계 힌트 없음 | 그림 단서 없음 (뜻은 정답 팝업에서만 공개) |

> **[2026-06-11 변경 사유]** 기존 3단 페이딩(L1 실선+양쪽 단서 / L2 점선+한쪽 단서 / L3 무단서)에서 L1을 제거하고 2모드로 축소했다. L1·L2 모두 경계 위치가 보여 비계가 중복되고, L1은 정답이 사실상 주어져 학습 과제가 사라지는 과잉 비계였기 때문이다(시리즈 선례: 2_vowel_finder 비계 3→2단 축소와 일치). 내부 값은 fadingLevel 2(연습하기)·3(도전하기)을 유지하고 값 1은 폐기한다.

## 7. 콘텐츠 및 데이터 범위

### 7.1 도입 세트 — S3(순우리말 어휘망) 공유 쌍 6개

S3 `words.js`에 합성어 전체가 실재하는 단어만을 도입 세트로 한정한다(§5.5 검증 반영 사항 (a) — 어휘 사실 정정).

| 합성어 | 공유 형태소 | S3 등재 확인 | 조각1 | 조각2 |
|---|---|---|---|---|
| 빗방울 | 방울 | ✅ (`rain_raindrop`) | 비(빗) | 방울 |
| 솔방울 | 방울 | ✅ (`mountain_pinecone`) | 솔 | 방울 |
| 별빛 | 빛 | ✅ (`night_star`) | 별 | 빛 |
| 달빛 | 빛 | ✅ (`night_moon`) | 달 | 빛 |
| 꽃송이 | 송이 | ✅ (`meadow_flower`) | 꽃 | 송이 |
| 눈송이 | 송이 | ✅ (`winter_snowflake`) | 눈 | 송이 |

**공유 형태소 페어링**: 도입 세트는 의도적으로 같은 형태소를 공유하는 쌍(방울/빛/송이 각 2개)으로 구성한다. 동일 조각의 재출현을 아동이 자연스럽게 관찰하되, 명시적 "모으기" 미션은 부여하지 않는다(Stage 4 payoff 보호).

**신규 자산 정직 등재**: 도입 세트의 합성어 전체는 S3에 실재하나, 분해된 조각 단위(비, 방울, 솔, 별, 빛, 달, 꽃, 송이, 눈) 각각의 독립 그림·TTS는 **소량 신규 자산**으로 제작이 필요하다. D1 미세 상승(어휘 0%→조각 그림 신규)을 인정한다.

### 7.2 잘못된 분절 예시 (오답 피드백용)

| 합성어 | 잘못된 분절 예 | 피드백 |
|---|---|---|
| 빗방울 | 빗방 + 울 | "그 조각은 뜻이 없네" + 복귀 |
| 별빛 | 별빛(미분절) | 경계 탭 안내 강조 |
| 꽃송이 | 꽃송 + 이 | "그 조각은 뜻이 없네" + 복귀 |

### 7.3 확장 세트 (선택적, 후속 구현)

도입 6개를 완전 숙달한 후 추가 가능한 순우리말 합성어 후보. 단, 모두 S3에 포함되거나 아동에게 친숙한 어휘여야 하며, 조각 양쪽이 독립적 의미를 가져야 한다.

| 후보어 | 조각1 | 조각2 | 비고 |
|---|---|---|---|
| 봄비 | 봄 | 비 | 계절+날씨 |
| 들꽃 | 들 | 꽃 | 자연 |
| 눈밭 | 눈 | 밭 | 겨울 |
| 봄볕 | 봄 | 볕 | 계절 |

| 잔존 도약 명시 | 본 게임의 조각은 자립·투명 합성어 형태소이고, Stage 6은 bound·불투명 한자 형태소를 다룬다. 이전되는 것은 "낱말=부품" 상위 통찰뿐이며, 한자 기호·Sino-Korean 의미 연결은 Stage 6에서 신규 획득한다 |
| D3 잔존 도약 처방 | 돋보기·핀치/가로 UI는 본 게임이 담당하지 않는다. Stage 6(형태소 탐정) Level 0 In-stage 온보딩이 D3를 완충해야 3→4 전이가 완전히 메워진다 |
| 데이터 스키마 | 분해 세트(합성어·조각1·조각2·그림 URL) JSON 스키마를 Stage 6가 참조 가능한 형태로 설계 권장 |

### 10.3 시리즈 전체 공통

- **PWA 독립성**: 게임별 고유 `CACHE_VERSION` 및 localStorage 접두사(`compound_split_*`) 사용
- **단일 학습자 프로필 DB**: 루트 AGENTS.md §Cross-Project Architecture에 따라 미래 통합 예정. 현 단계에서는 localStorage 단독 사용
- **SRL 게이팅**: 본 게임의 완료 기록이 Stage 4 진입 게이팅 조건으로 기능하도록 향후 확장 가능(현재는 권고 사항)

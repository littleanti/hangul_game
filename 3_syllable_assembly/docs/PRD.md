# 📋 PRD — 공감각적 음절 조립 게임

> Product Requirements Document
> Last updated: 2026-05-05
> Status: **구현 완료** — M7 실기기 테스트 대기

## 1. 제품 개요

**제품명**: 음절 조립소 (Syllable Assembly) — 자모를 모아 글자를 만들어요

**한 줄 요약**: 흩어진 한글 자모(초성·중성·종성) 블록을 시각적 은유 공간(냄비, 우주선, 마법 가마솥)에 끌어다 놓아 음절을 조립하는 모바일 우선 파닉스 게임.

**배포 환경**: 단일 페이지 정적 웹앱 + PWA 매니페스트 (iOS/Android 홈 화면 추가, 가로 잠금)

**상위 로드맵**: `1_chosung_quiz`(앵커 1, 음운 인출) → **`2_syllable_assembly`(자모 결합)** → `3_word_network`(어휘 확장)

## 2. 타겟 사용자

| 사용자 그룹 | 특징 | 주요 니즈 |
|---|---|---|
| 한글 첫 학습자 (만 4~6세) | 자모는 알지만 결합 원리 미숙, 손 정밀도 낮음 | 큰 터치 타겟, 즉각적 청각 피드백, 실패에 관대한 UX |
| 유치원·어린이집 교사 | 그룹 학습 도구 필요 | 가로 모드(태블릿 거치), 진척도 가시화 |
| 부모 | 자녀 자율 학습 | 안전(외부 통신 없음), 짧은 세션, 회전 잠금 |
| 한국어 입문 외국인 | 자모 형태와 음가 연결 | 자모-음소 매핑 시청각 동기화 |

## 3. 사용자 시나리오

### S1. 4세 아이가 태블릿으로 처음 시작
- 첫 화면 "🚀 시작" 큰 버튼 탭 → 가로 모드 안내 → 자동 회전 후 게임 진입
- 화면 가운데 우주선 도크가 나타나고, 하단에 자음·모음 블록이 흩어져 있음
- 빨간 `ㄱ`을 도크 좌측에 끌어다 놓자 `/그/` 발음 출력
- 파란 `ㅏ`를 우측에 놓으니 두 자모가 합쳐져 `가` 표시 + `/가/` 발음
- 노란 `ㅂ`을 하단에 놓아 `갑` 완성 → 우주선 점등 애니메이션 + 박수 효과음

### S2. 6세 아이가 리듬 모드에 도전
- 메인 메뉴에서 "🥁 리듬 모드" 선택
- 4/4박자 배경음악 시작, 자모가 화면 위에서 떨어짐
- 비트(♩ ♩ ♩ ♩)에 맞춰 정답 위치에 놓을 때마다 점수 가중치 +50%
- 콤보 5회 달성 시 무지개 트레일 효과

### S3. 교사가 가르치는 자모 범위 한정
- 설정에서 "오늘의 자모"만 선택 (예: ㄱ, ㄴ, ㄷ + ㅏ, ㅣ)
- 학습자에게 태블릿 전달 → 한정된 풀에서만 문제 출제
- 종료 화면에서 정답률·평균 조립 시간 확인

### S4. 부모가 짧게 끊어 학습
- "5문제만" 선택 → 약 3분 세션 후 자동 종료
- 부모가 "잘했어요!" 격려와 함께 태블릿 회수

## 4. 기능 요구사항

### P0 (필수, MVP)
| ID | 기능 | 설명 |
|---|---|---|
| F1 | 시작 화면 | "🚀 바로 시작" / "⚙️ 설정하고 시작" — 큰 버튼 (≥ 64dp) |
| F2 | 가로 모드 잠금 | 세로 감지 시 회전 안내 일러스트 표시 |
| F3 | 자모 도크 | 화면 하단 자음(빨강)·모음(파랑)·받침(노랑) 색상 구분 블록 |
| F4 | 시각적 은유 캔버스 | 우주선 / 냄비 / 가마솥 중 1개 테마 — 자모를 받는 슬롯 시각화 |
| F5 | 드래그 + 자성 스냅 | 정답 슬롯 ±20dp 자동 흡착 (정밀도 보완) |
| F6 | 모음 형태 분기 | 수직 모음(ㅏㅓㅣ)·수평 모음(ㅗㅜㅡ) 별 슬롯 자동 재배치 |
| F7 | 즉시 청각 피드백 | 자모 단독 발음 → 결합 직후 음절 발음 (`/ㄱ/+/ㅏ/=/가/`) |
| F8 | 음절 완성 보상 | 행성 점등·생명체 변환 등 짧은 애니메이션 (1.5초 이내) |
| F9 | 잘못된 조합 거부 | 부정적 피드백 최소화 — 블록이 부드럽게 튕겨 나감 |
| F10 | 진행률 표시 | 현재/총 문제, 별 스티커 누적 |
| F11 | 종료 화면 | 별 개수, 격려 메시지, "다시 하기" |

### P1 (확장)

### 추가 구현 (구현 완료)
| 기능 | 설명 | 상태 |
|---|---|---|
| 교정 모드 (correctionMode) | 자모를 슬롯에 올릴 때 틀리면 즉시 reject-flash 애니메이션 + 원위치. OFF 시 모든 슬롯 채운 후 일괄 검사 + wrong-shake | ✅ 완료 |
| 레벨 시스템 (1~4) | L1: 받침 없음 / L2: 쌍자음(85:15 비율) / L3: 홑받침 / L4: 겹받침(50:50 비율). 홈 화면 레벨 버튼으로 즉시 선택 | ✅ 완료 |
| 라운드 수 선택 | 5 / 10 / 15 / 20문제 선택 가능 (COUNT_OPTIONS) | ✅ 완료 |

### P2 (향후)
- 발음 평가 (Web Speech API + Levenshtein)로 학습자 발화 채점
- IndexedDB 기반 누적 데이터 + SRL(간격 반복) 큐
- 수집 도감(완성한 음절 컬렉션, 받침 결합 단어로 확장)
- 부모 대시보드(웹) — 학습 시간·정답률 가시화
- 다크 모드 + 폰트 크기 조절 (접근성)

## 5. 비기능 요구사항

| 항목 | 요구사항 |
|---|---|
| 성능 | 보급형 안드로이드(2GB RAM)에서 60fps 유지, 자모 결합 → 음절 표시 < 100ms |
| 호환성 | iOS Safari 15+, Chrome 100+ (Android), Edge 최신 |
| 디바이스 | iPad Mini 이상, 갤럭시 탭 A8 이상 1순위 / 7"+ 태블릿 / 5"+ 폰 |
| 화면 모드 | **가로(Landscape) 잠금** — 2D 자모 배치 영역 확보 |
| 터치 타겟 | **최소 64×64dp** (Apple HIG/Material 표준 48dp보다 크게) |
| 접근성 | 색맹 대응(자모 색상 + 형태 패턴 병기), 색상 대비 WCAG AA |
| 오프라인 | 초기 로드 후 네트워크 없이 작동 (PWA Service Worker) |
| 안전 | 외부 서버 통신 없음, 광고 없음, 회원가입 없음 |
| 자동재생 정책 | iOS Safari 우회 — 첫 화면 "시작" 탭 후 AudioContext 활성화 |
| 입력 안정성 | `touch-action: none` 으로 스크롤 충돌 방지, 더블탭 줌 비활성 |

## 6. 성공 지표

- **학습 효과**: 세션당 평균 7개 음절 조립 완료
- **이해도 전이**: 받침 없는 음절 → 받침 포함 단계 진입율 50%+
- **재사용**: 한 번 이상 "다시 하기" 비율 50%+ (4-6세 집중력 고려)
- **자율성**: 부모 개입 없이 5문제 완주 비율 60%+
- **모바일 안정성**: iOS·Android 양쪽에서 회전·자동재생 이슈 0건

## 7. 범위 제한 (Out of Scope)

- 회원가입/로그인 — 없음, 모든 것은 로컬
- 외부 서버로 데이터 전송 — 없음
- 멀티플레이어, 채팅
- 결제·인앱 구매
- 데스크톱 마우스 키보드 전용 UI(보조 환경으로만 동작)
- 한자 학습 (4단계 이후 게임에서 다룸)
- 단어 수준 학습 (3단계 게임의 영역)

## 10. 홈·설정·완료 화면 디자인 공통 규격

시리즈 전체의 통일감을 위해 **홈 화면(시작 화면)**, **설정 화면**, **게임 완료 화면(end-screen)** 은 `1_chosung_quiz` 의 디자인 시스템을 기준으로 제작한다.

| 디자인 요소 | 규격 |
|---|---|
| 제목 폰트 | **Jua** (Google Fonts) |
| 설명·본문 폰트 | **Gowun Dodum** (Google Fonts) |
| 버튼 디자인 | 1단계 `components.css` 버튼 스타일 — 큰 라운드 버튼, CSS 변수 색상 |
| 색감 | `1_chosung_quiz/src/css/tokens.css` CSS 변수와 동일한 팔레트 사용 |
| 배경 | `--color-bg` 등 공통 토큰과 동일한 배경 색상 |

> 플레이 화면(자모 조립 캔버스) 등 게임 고유 콘텐츠 화면은 각 게임 특성에 맞게 확장 가능하다.  
> 단, 시작·설정·완료 화면은 공통 디자인 언어를 유지한다.

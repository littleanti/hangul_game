# 📋 PRD — 한자 뿌리 역분해 게임

> Product Requirements Document
> Last updated: 2026-06-12
> Status: **구현 완료** — M0~M5 완료, M6 점진 변환 UX 개편(한자 1개 단위 즉시 판정·음절 제자리 변환) 반영

## 1. 제품 개요

**제품명**: 한자 뿌리 역분해 (Reverse Root Decomposition) — 합성어를 쪼개, 한자 뿌리를 찾아라

**한 줄 요약**: 친숙한 2형태소 합성어(화산=火+山, 동물=動+物)를 힌트 하에서 구성 한자로 분해·확인하는 역방향 분해 훈련 게임. 정방향 파생(1→N)은 인접 단계(6_morpheme_detective)가 담당하므로, 이 게임은 **역방향 분해(D→부품)만** 격리·훈련한다.

**핵심 인지 효과**: 합성어를 구성 형태소(한자)로 역분해하는 조합적 분석 능력 획득 → `8_vocabulary_tree`의 5~6형태소 학술어 분해(이등변삼각형=二+等+邊+三+角+形) 보스 스테이지를 무리 없이 진입하게 하는 전이 비계(scaffolding).

**배포 환경**: 단일 페이지 정적 웹앱 + PWA 매니페스트 (iOS/Android 홈 화면 추가)

**개발 포트**: 4330

**상위 로드맵**: `6_morpheme_detective`(형태소 탐정, 정방향 인식) → **`7_reverse_root`(역방향 분해)** → `8_vocabulary_tree`(어휘력 세계수, 학술어 파생)

## 2. 문제 정의 — 이 게임이 메우는 학습 갭

### 2.1 LEARNING_PROGRESSION_ANALYSIS.md §5.4 인용 근거

`LEARNING_PROGRESSION_ANALYSIS.md` §3.4(전이 4→5 분석)와 §5.4(`4.5_reverse_root` 재설계 제안)는 다음 갭을 진단한다.

> "본래 `4.5_root_sprout`(정방향 1→N 파생)는 **폐기** — 정방향은 S4 형제카드 덱이 이미 하는 일이라 redundancy 실패(세 비판 만장일치). 진짜 신규인 **역방향 분해**를 표적으로 재설계."
> — §5.4

전이 4→5(현 로드맵 기준 6_morpheme_detective → 8_vocabulary_tree) D2(인지 조작) 분석:

| 항목 | 내용 |
|------|------|
| 진단된 갭 차원 | **D2 인지 조작**: 정방향 유추는 `6_morpheme_detective` 형제카드가 이미 훈련. 진짜 신규는 **역방향 분해** 단 하나 |
| D5 선행 공백 | `6_morpheme_detective` 출구에 역분해 훈련 전무 — `8_vocabulary_tree` 학술어 보스 직면 시 인지 벼랑 |
| 처방 | 투명 2형태소 합성어를 힌트 하에 분해·확인하는 pre-boss 온보딩 — **7_reverse_root** |

### 2.2 정방향 폐기 근거

`6_morpheme_detective`의 핵심 메커닉(형제카드 덱·cross-word recurrence)은 한자 1자 → 다수 어휘 파생을 이미 반복 훈련한다. 정방향(1→N)을 이 게임에서 다시 훈련하면 학습 중복(redundancy)이 발생하며 ZPD 진전이 없다. 따라서 이 게임은 역방향(합성어→구성 한자) **단일 방향**만 다룬다.

| 초등 1~2학년 담임교사 | 국어·사회·과학 어휘 심화 단원 보조 도구 | 교과 연계 한자어 분해, 직관적 조작 |

### 인지 발달 정합

| 이론 | 정합 근거 |
|------|-----------|
| Piaget 구체적 조작기(7~11세) | 구체적 합성어(화산, 동물)를 물리적 조각으로 분리하는 직관적 조작이 이 시기 인지 구조에 최적 |
| Vygotsky ZPD | `6_morpheme_detective` 졸업 기술(형태소 재인·정방향 연결)을 비계로, 힌트 지원 역분해(ZPD 상단)에 도달 가능 |
| CLT 인지부하 이론 | 2형태소 투명 합성어만 사용해 요소 상호작용성 최소화. 힌트 3단 페이딩으로 외재적 부하 점진 감소 |
| Carlisle — 형태론적 인식 발달 | 투명 합성어(뜻이 직관적으로 드러나는 어휘)에서 형태소 인식이 먼저 발달하므로 진입 재료로 최적 |

### 4.1 획득 목표 (이 게임에서 달성)

| # | 획득 기술 | 설명 |
|---|-----------|------|
| G1 | 역방향 분해 조작 | 2형태소 투명 합성어를 구성 한자 2개로 분해하고, 각 한자의 뜻을 확인하는 능력 |
| G2 | 힌트 의존 역분해 자동화 | 3단 힌트 페이딩(뜻 라벨+하이라이트 → 하이라이트만 → 제거) 아래에서 정확한 분해 수행 |
| G3 | 투명 합성어 구조 통찰 | "낱말은 뜻을 지닌 한자 조각(형태소)의 결합"이라는 명제를 역방향에서 체득 |
| G4 | 7·8급 한자 재확인 | `6_morpheme_detective`에서 노출된 한자를 역분해 맥락에서 재인·재확인 |

| 비목표 항목 | 유보 단계 | 근거 |
|---|---|---|
| 무힌트(자유) 역분해 — 힌트 없이 스스로 분해 | `8_vocabulary_tree` 보스 스테이지 | 힌트 제거는 D2 추가 도약; 이 게임은 힌트 페이딩까지만 |

| SRL(간격 반복 학습) 스케줄링 | `8_vocabulary_tree` | 단기 집중 pre-boss 모듈 성격; SRL 오버헤드 불필요 |
| level-6/5 신규 한자(知·能·種·爭·質·敬·練·馬 등) 도입 | `8_vocabulary_tree` | 신규 어휘 부하 최소화 원칙 — 친숙어·기지 한자만 |
| 정방향 파생(1자 → N어휘 유추) | `6_morpheme_detective` (이미 완료) | redundancy — 형제카드 덱이 이미 훈련 |
| 한자 필순·쓰기 산출 | 별도 쓰기 산출 단계(미구현) | IME 회피 원칙·인코딩 단계 미설계 |
| 보스급 비투명 합성어(오불관언, 청출어람 등) | `10_literacy_decoder` 이후 | 표면형 추론 불가 어휘는 어휘 공백 발생 |

## 5. 핵심 게임 루프

```
[라운드 시작]
    │
    ▼
합성어 카드 제시 (예: "화산", TTS 발화, 음절 단위 렌더링)
    │
    ▼
힌트 레벨 표시 (L1: 뜻 라벨 + 색 하이라이트)
    │
    ▼
하단 도크에서 구성 한자 블록을 1개씩 탭/드래그로 선택 — 즉시 판정
    │
    ├─ 정답 한자 ──▶ 카드의 해당 음절이 한글→한자 제자리 변환
    │                 (예: 화산 → 火산, 한자 위 한글 루비 유지)
    │                 음·뜻 TTS + 정답 효과음, 블록 비활성
    │                 │
    │                 ├─ 남은 음절 있음 ──▶ [계속 선택]
    │                 │
    │                 └─ 전부 변환(火山) ──▶ 음·뜻 확인 팝업 + TTS
    │                                        긍정 피드백(효과음 + 별)
    │                                        ▼
    │                                  [다음 라운드 / 힌트 레벨 자동 하향]
    │
    └─ 오답 한자 ──▶ 해당 블록만 shake (피드백 귀속 명확화)
                     힌트 레벨 유지
                     ▼
                 [재시도]

[라운드 클리어]
    │
힌트 페이딩 스테이지 3단 진행:
  L1 뜻라벨+하이라이트 → L2 하이라이트만 → L3 힌트 없음(이 게임 천장)
    │
    ▼
[세션 완료] → 완료 화면 (별·성취 요약)
```

### 메커닉 핵심 원칙

| 원칙 | 내용 |
|------|------|
| 입력 방식 | `6_morpheme_detective` 계승 — 탭(발견)+자성 스냅(40dp) 드래그. IME 타이핑 없음 |
| 판정 단위 | **한자 1개 단위 즉시 판정** — 음절↔한자 1:1 대응이라 조합 간 상호작용이 없으므로 동시 제출의 인지 이득이 없음. 블록별 피드백 귀속(어느 선택이 틀렸는지)을 명확화 (CLT 외재 부하 감소) |
| 점진 변환 | 정답 한자마다 카드 음절이 한글→한자 제자리 변환(한글 루비 병기) — G3("낱말은 형태소 결합") 통찰의 직접 시각화 + 마이크로 보상 루프 |
| 힌트 페이딩 | 3단계 페이딩은 난이도가 아니라 **비계 감소** — 정답 구조 자체는 불변 |
| 빈칸 수 | 항상 **2형태소 합성어** (분해 조각 2개 고정) — 다중 부하 차단 |
| 어휘 선택 기준 | 친숙도 우선(초1~2 교과서 어휘 범위), `6_morpheme_detective` 기지 한자 100자 내 |
| 보스 없음 | 무힌트 보스 미션 없음 — `8_vocabulary_tree` pre-boss 온보딩이므로 클리맥스는 다음 단계에 귀속 |

## 7. 콘텐츠 및 데이터 범위

### 7.1 어휘 선정 기준

| 기준 | 내용 |
|------|------|
| 투명성 | 각 형태소의 뜻이 합성 결과에 직관적으로 반영되는 어휘만 (예: 화산=불+산, 동물=움직이는+것) |
| 한자 범위 | `6_morpheme_detective` 기지 한자 100자(한국어문회 7·8급) 내 |
| 어휘 친숙도 | 초등 1~2학년 교과서 어휘 범위(생활·과학·사회 기초어) |
| 신규 한자 0 | 이 게임에서 처음 등장하는 한자 없음 — 기지 100자 재사용만 |
| 1회 세션 분량 | 10~15개 합성어 (세션당 10~15분 내 완료 목표) |

| 소방(消防) | 消+防 | 소(꺼질)+방(막을) | 7급 기지 한자 |

| 인구(人口) | 人+口 | 인(사람)+구(입·인구) | 8급 기지 한자 |

> 최종 어휘 세트는 TRD 단계에서 `6_morpheme_detective/src/data/hanja.js` 및 `vocab.js`를 기준으로 교차 검증하여 확정한다.

### 7.3 힌트 3단 페이딩 구조

| 레벨 | 힌트 내용 | 제공 정보 |
|------|-----------|-----------|
| L1 (입문) | 각 형태소 영역 색 하이라이트 + 뜻 라벨 ("불 화" / "뫼 산") | 분해 경계 위치 + 각 한자의 음·뜻 |
| L2 (중급) | 색 하이라이트만 (뜻 라벨 없음) | 분해 경계 위치만 |
| L3 (고급, 이 게임의 천장) | 힌트 없음 — 합성어 카드만 표시 | 없음 (단어 자체만) |

> L3 통과 = `7_reverse_root` 졸업 기준. 무힌트 보스 미션(불투명 복합어·5형태소 이상)은 `8_vocabulary_tree`에 귀속.

### 7.4 오답 디스트랙터 설계

각 라운드의 한자 블록 도크에는 정답 2개 + 디스트랙터 2~3개가 포함된다.

| 디스트랙터 유형 | 예시 (화산 라운드) | 출현 의도 |
|---|---|---|
| 음독 유사 한자 | 花(꽃 화) — 火와 음독 동일 | 음독 의존 분해 오류 유발, 의미 기반 분해 유도 |
| 의미 근접 한자 | 岩(바위 암) — 山과 의미 유사 | 개념 연상 오류 차단 |
| 무관 기지 한자 | 水(물 수) | 무작위 도크 채우기 |

## 8. 성공 지표

| 지표 | 목표값 | 측정 방법 |
|------|--------|-----------|
| 세션 완주율 | ≥ 75% | 세션 시작 대비 완료 화면 도달 비율 |
| L3(무힌트) 정답률 | ≥ 70% | L3 라운드 정답 수 / L3 총 시도 수 |
| 1회 이상 재시작 비율 | ≥ 35% | "다시 하기" 탭 횟수 |
| 평균 세션 시간 | 8~15분 | 시작~완료 화면 경과 시간 |
| `8_vocabulary_tree` 첫 보스 정답률 | ≥ 60% | 이 게임 졸업자 추적(단계 간 프로파일 게이팅 구현 시) |

> 주의: 현재 로드맵에 cross-stage SRL/프로파일 게이팅이 미구현(`LEARNING_PROGRESSION_ANALYSIS.md` §4 §5번 공백)이므로, 마지막 지표는 게이팅 구현 이후 추적 가능.

## 9. 시리즈 연속성 요구

### 9.1 이전 단계(`6_morpheme_detective`)와의 연결

| 항목 | 내용 |
|------|------|
| 데이터 계승 | `6_morpheme_detective/src/data/hanja.js` 100자(7·8급) + `vocab.js` 429어휘를 어휘 풀의 출발점으로 활용. 신규 한자 추가 없음 |
| 메커닉 계승 | 탭-발견 + 자성 스냅(40dp) 드래그 패러다임 직접 계승 — 조작 패러다임 변경 없음(D3 동결) |
| 어휘 재사용 | `6_morpheme_detective` 사건 10건 어휘(주차장·교실·가족 집 등)에서 2형태소 투명 합성어를 우선 선별 |
| 선행 기술 전제 | 한국어문회 7·8급 한자 100자 재인, 형태소 인식 진입, 단어의 분해 가능성 인지 |
| localStorage 키 분리 | `6md:` (morpheme detective) 스토리지와 충돌 없도록 `7rr:` 접두사 사용 |

### 9.2 다음 단계(`8_vocabulary_tree`)와의 연결

| 항목 | 내용 |
|------|------|
| 출구 기술 | L3(무힌트) 역분해 정확률 ≥ 70% → `8_vocabulary_tree` 진입 준비 완료 신호 |
| 어휘 연속성 | 이 게임 졸업 어휘(투명 2형태소 합성어 세트)는 `8_vocabulary_tree` 초기 라운드의 워밍업 재료로 재출현 권장 |
| 난이도 계단 | 이 게임(2형태소·투명·힌트 하) → `8_vocabulary_tree`(5~6형태소·반투명~불투명·무힌트 보스). 단 한 축씩 전진 |
| 유보된 기술 인수 | 이 게임이 명시적으로 유보한 항목(무힌트 보스·SRL·level-6/5 신규 한자)을 `8_vocabulary_tree`가 정식 인수 |

### 9.3 전체 시리즈 위상

```
6_morpheme_detective  →  7_reverse_root  →  8_vocabulary_tree
   (정방향 인식)           (역방향 분해)        (파생+학술어 분해)
   탭-발견+자성 스냅  ──계승──▶  동일 패러다임  ──비계──▶  세로 드래그+카드 슬롯
   7·8급 한자 100자   ──재사용──▶  동일 풀(0 신규) ──확장──▶  level-6/5 신규 한자 추가
   대상: 초1~2(7~8세)          만 8세 전후             초3~4(8~10세)
```

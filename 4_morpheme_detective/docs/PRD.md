# 📋 PRD — 형태소 탐정 게임

> Product Requirements Document
> Last updated: 2026-05-25
> Status: **M9 한자 풀 100자 확장 + 사건 10건** — 한국어문회 7·8급 100자(8급 50 + 7급II 추가 50) 전체 데이터/도감 등록, 사건 10개(주차장·교실·가족 집·급식실·소방서·자연공원·시장·하늘·거리·숫자) 각 10자 분배, 어휘 429개 자동 매핑(`scripts/gen-vocab.mjs`), 한자 JSON 100개 placeholder + 일러스트 10장 grid placeholder, 도감 100칸(5/8/10열 반응형) + 급수 필터(전체/8급/7급Ⅱ, localStorage 영속), PWA SW v4 (145 URLs). 남은 작업: 실 일러스트 10장 손그림 / morph path 정식 자산(Make Me a Hanzi) / 실기기 매트릭스 / Noto Sans CJK 서브셋 / F18.

## 1. 제품 개요

**제품명**: 형태소 탐정 (Morpheme Detective) — 익숙한 단어 속 숨은 한자를 찾아라

**한 줄 요약**: 익숙한 일상 공간(급식실, 주차장, 소방서 등) 일러스트를 '돋보기'로 탐색해, 일상 어휘 속에 숨어 있는 한자(뜻글자)를 시각적·서사적으로 발견하게 하는 모바일 우선 학습 게임.

**핵심 인지 효과**: 단어의 분해 가능성 인지 → 음운(소리) ↔ 형태소(뜻) 통합 → 학습 어휘 폭발의 키가 되는 **형태소 인식(Morphological Awareness)** 진입.

**배포 환경**: 단일 페이지 정적 웹앱 + PWA 매니페스트 (iOS/Android 홈 화면 추가, 가로/세로 모드 지원)

**상위 로드맵**: `3_word_network`(고유어 어휘망, 자동 읽기) → **`4_morpheme_detective`(한자 형태소 인식)** → `5_vocabulary_tree`(1자 → N어휘 파생)

## 2. 타겟 사용자

| 사용자 그룹 | 특징 | 주요 니즈 |
|---|---|---|
| 초등 1 ~ 2학년 | 일상 어휘는 자동 읽기 가능, 한자 미학습 | 한자에 대한 부담 없이 시각적 발견의 즐거움 |
| 초등 저학년 학부모 | 자녀의 어휘력·문해력 기초 형성에 관심 | 강제 암기 없는 자연스러운 노출, 일상 연계 활동 |
| 초등 1·2학년 담임교사 | 국어 어휘 단원 보조 학습 도구 | 교과서 어휘와 한자 연계, 짧은 세션, 가로 태블릿 거치 |
| 한국어 학습 외국인(아동) | 한자어 어휘 분해 학습 | 직관적 시각 변형, 발음 자동 청취 |

## 3. 사용자 시나리오

### S1. 초1 아이가 태블릿으로 처음 시작
- 첫 화면 "🔍 탐정 시작" 큰 버튼 탭 → 게임 진입 (가로/세로 모두 지원, 회전 안내 없음)
- "오늘의 탐정 사건: 학교 급식실" 카드 등장 → 일러스트 진입
- 일러스트 내 객체들(자동차 그림, 표지판 등)이 처음 5초간 부드럽게 펄스
- 아이가 손가락으로 '주차장' 표지판 탭 → 단어가 "주" "차" "장" 음절 블록으로 분리
- '차' 음절이 노란 테두리로 하이라이트 + TTS "차"
- 옆 자동차의 수레바퀴 실루엣이 갑골문 → 해서체 한자 '車'로 부드럽게 변형
- 같은 한자를 공유하는 어휘 카드 4개 등장: 자동차, 기차, 자전거, 마차
- "잘했어요! 1단계 클리어" + 별 1개 적립

### S2. 초2 아이가 두 번째 한자 탐색
- 메인 메뉴에서 "🌊 다음 사건: 약수터" 선택
- '생수병', '약수터 표지판', '수영장 안내문' 등이 클릭 가능
- '수영장' 클릭 → "수" "영" "장" → '수' 하이라이트
- 흐르는 물결 일러스트가 한자 '水'로 변형
- 어휘 카드: 생수, 수영, 약수, 정수기 + 발음 자동 재생
- 종료 후 미션 카드: "오늘은 '水' 한자를 발견했어요. 집에 가는 길에 '수'가 들어간 단어를 함께 찾아보세요."

### S3. 교사가 단원 한자 한정 학습
- 설정에서 "오늘의 한자"를 `日, 月, 火, 水, 木` 5자로 한정 (요일 단원 연계)
- 학습자에게 태블릿 전달 → 한정된 풀에서만 사건 출제
- 종료 화면에서 발견한 한자 수, 평균 탐색 시간 확인

### S4. 부모가 짧은 학습 + 일상 연계
- "사건 1건만" 옵션 선택 → 약 4분 세션
- 종료 시 "함께 찾아보기" 미션 카드 인쇄/스크린샷 가능
- 부모가 등하굣길에 함께 한자 발견 놀이로 이어짐

## 4. 기능 요구사항

### P0 (필수, MVP)
| ID | 기능 | 설명 |
|---|---|---|
| F1 | 시작 화면 | "🔍 시작" / "⚙️ 설정" — 큰 버튼 (≥ 64dp) |
| F2 | 가로/세로 모드 지원 | 가로: 기존 풍경 레이아웃. 세로: 일러스트 상단 70%(letterbox) + 단어블록·카드 하단 30%. 회전 안내 없음. |
| F3 | 사건(스테이지) 카드 선택 | 학교 급식실·주차장·소방서·약수터 등 1 ~ 2개 일러스트 |
| F4 | 일러스트 내 클릭 가능 객체 | 표지판/간판/사물 SVG hit zone, 최소 80×80dp |

| F6 | 돋보기 인터랙션 | 손가락/마우스 위치 추적 + 자석 흡착(가까운 hit zone에 끌림) |
| F7 | 단어 → 음절 분리 애니메이션 | 클릭 단어가 좌→우 음절 블록으로 분해 |
| F8 | 핵심 한자 음절 하이라이트 | 색상 테두리 + TTS 음·뜻 재생 |
| F9 | 상형문자 변형 애니메이션 | 객체 실루엣 → 갑골문 → 해서체 한자 (SVG path morph 우선, 1.5 ~ 2.5초) |
| F10 | 한자 공유 어휘 카드 | 같은 한자가 들어간 친숙한 어휘 3 ~ 5개 표시 + 탭 시 발음 + **탭 시 위 음절 블록을 해당 단어로 갱신**(같은 한자가 다른 단어 어느 음절에 위치하는지 시각적 비교) |
| F11 | 진행률 + 별 스티커 | 발견한 한자 수, 별 누적 |
| F12 | 종료 화면 + 미션 카드 | "오늘은 'X' 한자를 발견했어요" 일상 연계 미션 |

### P2 (향후)
- IndexedDB 기반 SRL(간격 반복) 큐 — 한자별 재노출 스케줄
- 발견 한자 → 5단계(`5_vocabulary_tree`) 학습 큐 자동 전달
- 부모 대시보드(웹) — 발견 한자 수·취약 한자·세션 시간 가시화

- 발음 평가(Web Speech API + Levenshtein)로 어휘 카드 따라 읽기 채점
- 일러스트 자동 생성 보조(저작 도구 — 운영 사이드)

## 6. 성공 지표

- **학습 효과**: 세션당 평균 2개 한자 발견, 한 한자당 어휘 카드 평균 3개 이상 탐색
- **이해도 전이**: 미션 카드 발급 후 1주 내 학습자가 일상에서 같은 한자 단어를 자발적으로 찾는 사례 (학부모 인터뷰 기준 30%+)
- **재사용**: 한 번 이상 "다른 사건" 진입 비율 50%+
- **자율성**: 부모 개입 없이 1개 사건 완주 비율 70%+ (초1 ~ 초2 집중력)
- **모바일 안정성**: SVG 변형 애니메이션 30fps+ 유지, 자동재생/회전 이슈 0건
- **거부감 없음**: 사용자 테스트에서 "한자가 어렵다" 피드백 < 10% (게임 내 한자 쓰기 입력 부재 정책 효과 검증)

## 7. 범위 제한 (Out of Scope)

- 회원가입/로그인 — 없음, 모든 것은 로컬
- 외부 서버로 데이터 전송 — 없음
- **한자 쓰기 입력** — 절대 금지 (인지 부담, 본 게임의 핵심 정책)
- 새로운 단어로 한자 노출 — 학습자가 **이미 아는 단어**에서만 출발
- 멀티플레이어, 채팅
- 결제·인앱 구매
- 음운 인식 학습(2단계 게임의 영역)
- 어휘 단순 확장(3·5단계 게임의 영역)
- 문맥 추론·직독직해(6단계 게임의 영역)

## 8. 의존 게임 / 데이터 호환

- **선행**: `3_word_network` — 일상 어휘 자동 읽기 졸업 가정 (어휘 자체는 이미 안다)
- **후행**: `5_vocabulary_tree` — 발견한 한자를 줄기로 어휘 가지를 뻗는 게임으로 자연스럽게 연결
- **공유 후보**:
  - 어휘 DB: `1_chosung_quiz/src/data/words.js` 형식 확장 — 어휘에 한자 메타(`hanja: ['車']`) 추가
  - 학습자 진척도 스키마 — 7단계 공통 설계, 본 게임에서 발견 한자 컬렉션 키(`4md:collected`) 추가
  - 한자 매핑 DB(8 ~ 7급 약 100자) — `5_vocabulary_tree` 와 동일 자산 공유

## 10. 홈·설정·완료 화면 디자인 공통 규격

시리즈 전체의 통일감을 위해 **홈 화면(시작 화면)**, **설정 화면**, **게임 완료 화면(end-screen / mission-screen)** 은 `1_chosung_quiz` 의 디자인 시스템을 기준으로 제작한다.

| 디자인 요소 | 규격 |
|---|---|
| 제목 폰트 | **Jua** (Google Fonts) |
| 설명·본문 폰트 | **Gowun Dodum** (Google Fonts) |
| 버튼 디자인 | 1단계 `components.css` 버튼 스타일 — 큰 라운드 버튼, CSS 변수 색상 |
| 색감 | `1_chosung_quiz/src/css/tokens.css` CSS 변수와 동일한 팔레트 사용 |
| 배경 | `--color-bg` 등 공통 토큰과 동일한 배경 색상 |

> 플레이 화면(가로/세로 일러스트·돋보기 인터랙션) 등 게임 고유 콘텐츠 화면은 각 게임 특성에 맞게 확장 가능하다.  
> 단, 시작·설정·완료 화면은 공통 디자인 언어를 유지한다.
